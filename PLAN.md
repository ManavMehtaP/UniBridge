## Master Context

### What This Product Is

UniPortal is a multi-tenant SaaS university management platform with three actor roles вЂ” **HOD**, **Faculty**, and **Student** вЂ” operating under a single university tenant. It handles academic years, semester cycles, batch/division management, result uploads, attendance, notes, mentor-student chat, and AI-powered study tools.

A single university can have multiple HODs (one per batch group), multiple faculties, and hundreds of students вЂ” all under the same tenant subdomain (e.g., `lju.uniportal.in`).

---

### Critical Domain Rules (Read Before Touching Any Code)

1. **enrollment_no is the permanent student identity** вЂ” generated by the college at admission, never changes across 4 years. This is the join key everywhere.

2. **roll_no changes every semester** вЂ” it is a per-semester assignment stored inside `StudentEnrollment`, NOT on the `Student` table.

3. **Batch/Division changes every semester** вЂ” e.g., Kavy can be in `B3` in sem-1, then `B1` in sem-2, then `C2` in sem-3 and sem-4. The batch is always a property of the enrollment row, not the student.

4. **Year level progression is FY в†’ SY в†’ TY в†’ Final** вЂ” specifically: a student in `FY-3` can be promoted to `SY-2` (not necessarily SY-1). The target batch on promotion is decided by the HOD during the promotion wizard. The system does NOT auto-assign вЂ” it presents a mapping UI.

5. **HOD is also a Faculty** вЂ” the HOD row IS a faculty row with `is_hod = true`. HOD has no mentor students assigned. HOD manages data (CSV upload, results, subjects, academic calendar). HOD also has analytics access across all batches.

6. **Mentor code is a 3-character string** (e.g., `SYD`) assigned to a faculty. One faculty в†’ many students. Students see only their assigned mentor. The code is per-semester (a faculty can have a different code next semester).

7. **Phases T1, T2, T3, T4** вЂ” each semester has exactly 4 phases. After each phase there is a university exam. Results are uploaded per phase, per subject, by the HOD.

8. **Subject list is per-semester** вЂ” HOD defines subjects when creating a semester. Subjects are NOT shared across semesters automatically (a subject can have the same name but is a different row in a different semester).

9. **All historical data is preserved** вЂ” no hard deletes on academic records. Soft-delete with `deleted_at` or archive flags.

10. **Multi-tenant isolation** вЂ” every query is scoped by `university_id`. The Express middleware injects `req.university` from the subdomain before every route handler.

### Technology Stack

| Layer | Technology | Purpose |
|---|---|---|
| Frontend | React 18 + Vite + TypeScript | All three portals (HOD, Faculty, Student) |
| Routing | React Router v6 | Role-based protected routes |
| State | Zustand + React Query (TanStack) | Global auth state + server cache |
| UI | Tailwind CSS + shadcn/ui | Components |
| Backend API | Express + TypeScript | REST API, auth, file handling, websocket |
| ORM | Prisma | PostgreSQL schema + migrations |
| Database | PostgreSQL | Primary datastore |
| Cache | Redis | Sessions, presence, attendance % cache |
| File Storage | AWS S3 | Notes PDFs, result CSVs, profile photos |
| Real-time | Socket.io | Mentor-student chat, online presence |
| AI Service | Django + DRF | All AI features (separate service) |
| AI / LLM | LangChain + OpenAI / Ollama | LLM orchestration |
| Vector DB | pgvector (PostgreSQL ext) | RAG for notes + PYQ |
| Task Queue | Celery + Redis | Async AI jobs (PDF parsing, embeddings) |
| Auth | JWT (access + refresh) | Stateless auth across services |
| Deploy | Docker Compose в†’ AWS EC2 | Self-hostable per university or shared |
| CI | GitHub Actions | Lint, test, build, deploy |

### Prisma Schema (Complete)

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// в”Ђв”Ђв”Ђ TENANT в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ

model University {
  id         String   @id @default(uuid())
  name       String
  slug       String   @unique   // used as subdomain: lju.uniportal.in
  logoUrl    String?
  plan       Plan     @default(FREE)
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt

  academicYears  AcademicYear[]
  faculties      Faculty[]
  students       Student[]

  @@map("universities")
}

enum Plan {
  FREE
  PRO
  ENTERPRISE
}

// в”Ђв”Ђв”Ђ ACADEMIC YEAR в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ

model AcademicYear {
  id           String   @id @default(uuid())
  universityId String
  label        String   // "2024-25"
  isActive     Boolean  @default(false)
  startDate    DateTime
  endDate      DateTime
  createdAt    DateTime @default(now())

  university   University  @relation(fields: [universityId], references: [id])
  semesters    Semester[]
  batches      Batch[]

  @@unique([universityId, label])
  @@map("academic_years")
}

// в”Ђв”Ђв”Ђ SEMESTER в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// Semester number is global (1вЂ“8 across 4 years).
// sem 1вЂ“2 = FY, sem 3вЂ“4 = SY, sem 5вЂ“6 = TY, sem 7вЂ“8 = Final Year

model Semester {
  id             String   @id @default(uuid())
  universityId   String
  academicYearId String
  number         Int      // 1 to 8
  label          String   // "Semester 1", "Semester 3"
  yearLevel      YearLevel  // FY | SY | TY | FINAL
  isActive       Boolean  @default(false)
  startDate      DateTime
  endDate        DateTime
  phaseCount     Int      @default(4)  // always 4 (T1вЂ“T4)
  createdAt      DateTime @default(now())

  academicYear   AcademicYear  @relation(fields: [academicYearId], references: [id])
  subjects       Subject[]
  enrollments    StudentEnrollment[]
  phases         Phase[]
  timetableSlots TimetableSlot[]
  calendarEvents CalendarEvent[]

  @@map("semesters")
}

enum YearLevel {
  FY    // First Year
  SY    // Second Year
  TY    // Third Year
  FINAL // Fourth Year / Final Year
}

// в”Ђв”Ђв”Ђ PHASE (T1, T2, T3, T4) в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ

model Phase {
  id         String   @id @default(uuid())
  semesterId String
  label      String   // "T1", "T2", "T3", "T4"
  number     Int      // 1 to 4
  startDate  DateTime
  endDate    DateTime
  examDate   DateTime?

  semester   Semester  @relation(fields: [semesterId], references: [id])
  results    Result[]

  @@unique([semesterId, number])
  @@map("phases")
}

// в”Ђв”Ђв”Ђ BATCH / DIVISION в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// e.g., SY-3, C2, B1, FY-2
// A batch belongs to a year and a semester.

model Batch {
  id             String   @id @default(uuid())
  universityId   String
  academicYearId String
  code           String   // "SY-3", "C2", "B1", "FY-2"
  yearLevel      YearLevel
  createdAt      DateTime @default(now())

  academicYear   AcademicYear  @relation(fields: [academicYearId], references: [id])
  enrollments    StudentEnrollment[]
  facultyBatches FacultyBatchAssignment[]
  timetableSlots TimetableSlot[]

  @@unique([academicYearId, code])
  @@map("batches")
}

// в”Ђв”Ђв”Ђ SUBJECT в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// Subject list is defined per semester by HOD.

model Subject {
  id           String   @id @default(uuid())
  semesterId   String
  name         String   // "Computer Organization & Architecture"
  code         String   // "COA"
  credits      Int      @default(4)
  createdAt    DateTime @default(now())

  semester     Semester  @relation(fields: [semesterId], references: [id])
  results      Result[]
  notes        Note[]
  attendances  AttendanceRecord[]
  timetableSlots TimetableSlot[]
  facultyBatches FacultyBatchAssignment[]

  @@unique([semesterId, code])
  @@map("subjects")
}

// в”Ђв”Ђв”Ђ FACULTY в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// HOD is also a faculty with is_hod = true.
// HOD has no MentorAssignments.

model Faculty {
  id             String   @id @default(uuid())
  universityId   String
  name           String
  email          String   @unique
  passwordHash   String
  employeeId     String   // College-assigned employee ID
  department     String   // "IT", "CSE", "CE"
  isHod          Boolean  @default(false)
  isActive       Boolean  @default(true)
  profilePhotoUrl String?
  phone          String?
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
  deletedAt      DateTime?

  university         University  @relation(fields: [universityId], references: [id])
  batchAssignments   FacultyBatchAssignment[]
  mentorAssignments  MentorAssignment[]
  attendanceRecords  AttendanceRecord[]
  announcements      Announcement[]
  uploadedNotes      Note[]
  quizzes            Quiz[]
  chatMessagesAsFaculty ChatMessage[] @relation("FacultyChatMessages")
  refreshTokens      RefreshToken[]

  @@map("faculties")
}

// Maps which faculty teaches which subject in which batch in which semester

model FacultyBatchAssignment {
  id         String @id @default(uuid())
  facultyId  String
  batchId    String
  subjectId  String
  semesterId String

  faculty    Faculty  @relation(fields: [facultyId], references: [id])
  batch      Batch    @relation(fields: [batchId], references: [id])
  subject    Subject  @relation(fields: [subjectId], references: [id])

  @@unique([facultyId, batchId, subjectId, semesterId])
  @@map("faculty_batch_assignments")
}

// в”Ђв”Ђв”Ђ STUDENT в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// enrollment_no is the permanent college-assigned ID. Never changes across 4 years.
// roll_no changes each semester вЂ” stored in StudentEnrollment.

model Student {
  id              String   @id @default(uuid())
  universityId    String
  enrollmentNo    String   @unique  // e.g., "LJ20IT001" вЂ” permanent, college-issued
  name            String
  email           String   @unique
  passwordHash    String
  branch          String   // "IT", "CSE", "CE", "AIML", "RAI" вЂ” permanent
  admissionYear   Int      // e.g., 2023
  profilePhotoUrl String?
  phone           String?
  isActive        Boolean  @default(true)
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  deletedAt       DateTime?

  university          University  @relation(fields: [universityId], references: [id])
  enrollments         StudentEnrollment[]
  mentorAssignments   MentorAssignment[]
  selfNotes           SelfNote[]
  quizAttempts        QuizAttempt[]
  chatMessagesAsStudent ChatMessage[] @relation("StudentChatMessages")
  refreshTokens       RefreshToken[]

  @@map("students")
}

// в”Ђв”Ђв”Ђ STUDENT ENROLLMENT (PIVOT) в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// One row per student per semester. This is where roll_no and batch live.
// Querying a student's full journey = all their enrollment rows ordered by semester.

model StudentEnrollment {
  id             String   @id @default(uuid())
  studentId      String   // FK to Student (via enrollment_no internally)
  semesterId     String
  batchId        String
  rollNo         String   // changes every semester, e.g., "IT-23-042"
  yearLevel      YearLevel  // FY | SY | TY | FINAL
  promotedFromId String?  // FK to previous enrollment row (null for first sem)
  isCurrent      Boolean  @default(true)  // only ONE row per student is current
  createdAt      DateTime @default(now())

  student        Student   @relation(fields: [studentId], references: [id])
  semester       Semester  @relation(fields: [semesterId], references: [id])
  batch          Batch     @relation(fields: [batchId], references: [id])
  promotedFrom   StudentEnrollment?  @relation("PromotionHistory", fields: [promotedFromId], references: [id])
  promotedTo     StudentEnrollment[] @relation("PromotionHistory")
  results        Result[]
  attendances    AttendanceRecord[]

  @@unique([studentId, semesterId])
  @@map("student_enrollments")
}

// в”Ђв”Ђв”Ђ MENTOR ASSIGNMENT в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// One faculty (non-HOD) mentors many students.
// mentorCode is the 3-char code like "SYD" assigned to the faculty for this semester.
// HOD never appears in this table.

model MentorAssignment {
  id          String   @id @default(uuid())
  facultyId   String
  studentId   String
  semesterId  String
  mentorCode  String   // "SYD" вЂ” faculty's code for this semester
  createdAt   DateTime @default(now())

  faculty     Faculty  @relation(fields: [facultyId], references: [id])
  student     Student  @relation(fields: [studentId], references: [id])
  chatMessages ChatMessage[]

  @@unique([facultyId, studentId, semesterId])
  @@map("mentor_assignments")
}

// в”Ђв”Ђв”Ђ RESULT в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// Uploaded by HOD only. Per enrollment (student + semester), per phase, per subject.

model Result {
  id            String   @id @default(uuid())
  enrollmentId  String   // StudentEnrollment id
  phaseId       String   // T1 / T2 / T3 / T4
  subjectId     String
  marksObtained Float
  maxMarks      Float    @default(100)
  grade         String?
  uploadedById  String   // Faculty (HOD) id
  uploadedAt    DateTime @default(now())

  enrollment    StudentEnrollment  @relation(fields: [enrollmentId], references: [id])
  phase         Phase              @relation(fields: [phaseId], references: [id])
  subject       Subject            @relation(fields: [subjectId], references: [id])

  @@unique([enrollmentId, phaseId, subjectId])
  @@map("results")
}

// в”Ђв”Ђв”Ђ ATTENDANCE в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
// One row per lecture delivered. Faculty marks per-student presence.

model AttendanceRecord {
  id           String   @id @default(uuid())
  enrollmentId String
  subjectId    String
  facultyId    String
  lectureDate  DateTime
  isPresent    Boolean
  isLocked     Boolean  @default(false)  // HOD can lock after approval
  createdAt    DateTime @default(now())

  enrollment   StudentEnrollment  @relation(fields: [enrollmentId], references: [id])
  subject      Subject            @relation(fields: [subjectId], references: [id])
  faculty      Faculty            @relation(fields: [facultyId], references: [id])

  @@unique([enrollmentId, subjectId, lectureDate])
  @@map("attendance_records")
}

// в”Ђв”Ђв”Ђ NOTES в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ

model Note {
  id           String    @id @default(uuid())
  subjectId    String
  facultyId    String    // who uploaded
  title        String
  description  String?
  fileUrl      String    // S3 URL
  fileKey      String    // S3 key for deletion
  mimeType     String
  aiSummary    String?   // generated by Django AI service
  createdAt    DateTime  @default(now())
  deletedAt    DateTime?

  subject      Subject  @relation(fields: [subjectId], references: [id])
  faculty      Faculty  @relation(fields: [facultyId], references: [id])
  flashcards   Flashcard[]

  @@map("notes")
}

model Flashcard {
  id       String @id @default(uuid())
  noteId   String
  question String
  answer   String

  note     Note @relation(fields: [noteId], references: [id])

  @@map("flashcards")
}

model SelfNote {
  id        String   @id @default(uuid())
  studentId String
  subjectId String?
  title     String
  content   String   // rich text (JSON or HTML)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  student   Student @relation(fields: [studentId], references: [id])

  @@map("self_notes")
}

// в”Ђв”Ђв”Ђ QUIZ в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ

model Quiz {
  id           String    @id @default(uuid())
  facultyId    String
  subjectId    String
  semesterId   String
  title        String
  isAiGenerated Boolean  @default(false)
  isPublished  Boolean   @default(false)
  dueDate      DateTime?
  createdAt    DateTime  @default(now())

  faculty      Faculty    @relation(fields: [facultyId], references: [id])
  questions    Question[]
  attempts     QuizAttempt[]

  @@map("quizzes")
}

model Question {
  id            String   @id @default(uuid())
  quizId        String
  text          String
  options       Json     // [{ id, text }]
  correctOption String
  explanation   String?

  quiz          Quiz @relation(fields: [quizId], references: [id])

  @@map("questions")
}

model QuizAttempt {
  id          String   @id @default(uuid())
  studentId   String
  quizId      String
  score       Float
  answers     Json     // { questionId: selectedOptionId }
  submittedAt DateTime @default(now())

  student     Student @relation(fields: [studentId], references: [id])
  quiz        Quiz    @relation(fields: [quizId], references: [id])

  @@unique([studentId, quizId])
```

### Chat System (Socket.io)

**Backend вЂ” Socket.io Server (`socket/handlers/chat.handler.ts`)**

Socket auth: On `connection`, middleware reads `auth.token` from handshake, verifies JWT, attaches `socket.data.user`.

Events emitted by client:
- `join_room` в†’ `{ mentorAssignmentId }` в†’ server joins socket to room `mentor_chat:${mentorAssignmentId}`
- `send_message` в†’ `{ mentorAssignmentId, content }` в†’ server:
  1. Validates sender is part of this `MentorAssignment`
  2. Creates `ChatMessage` row in DB
  3. Emits `new_message` to room with full message object
- `mark_read` в†’ `{ mentorAssignmentId }` в†’ updates all unread messages where recipient is sender role

Events emitted by server:
- `new_message` в†’ `{ id, senderRole, content, sentAt, senderName }`
- `user_online` в†’ `{ userId, role }` (presence)
- `user_offline` в†’ `{ userId, role }`

Presence (Redis pub/sub):
- On socket connect в†’ `SETEX presence:{userId} 60 "online"` (TTL 60s, heartbeat renews)
- On disconnect в†’ `DEL presence:{userId}` + emit `user_offline` to relevant rooms
- Client sends heartbeat ping every 30s

**REST endpoints for chat history:**

`GET /chat/messages?mentorAssignmentId=&page=1&limit=30` вЂ” paginated message history, ordered `sentAt` desc

`GET /faculty/mentees/unread` вЂ” returns `[{ mentorAssignmentId, studentName, unreadCount }]` for sidebar badges

### Django Service Architecture

**Service runs on port 8001, internal only (not exposed to internet)**

**Django apps:**

`apps/notes/` вЂ” PDF parsing, summary generation, flashcard creation

`apps/quiz/` вЂ” MCQ generation from notes/topics

`apps/pyq/` вЂ” PYQ PDF analysis and topic frequency mapping

**Celery workers** handle all slow jobs (PDF parsing, embedding generation, AI generation). Django views enqueue jobs and return a `jobId`. Client polls `/ai/jobs/{jobId}/status` until done.

### Smart Notes (Django)

`POST /notes/process` вЂ” triggered by Express when faculty uploads a note
- Input: `{ noteId, fileUrl, subjectCode }`
- Worker: download PDF from S3 в†’ extract text via PyMuPDF в†’ chunk text в†’ embed chunks via OpenAI `text-embedding-3-small` в†’ store vectors in pgvector with `note_id` metadata
- Once embeddings done в†’ generate summary via LLM (prompt: "Summarize this lecture note in 300 words for a university student...")
- Generate 5вЂ“10 flashcards (Q&A pairs)
- POST back to Express: `PATCH /internal/notes/{noteId}` with `{ aiSummary, flashcards }`

Express stores summary on `Note.aiSummary` and creates `Flashcard` rows.

Student sees: Note card with "AI Summary" expandable section + "Flashcards" button.

---

### AI Assistant (Django RAG)

`POST /assistant/chat` вЂ” input: `{ messages: [...], universityId, studentId, subjectId? }`
- Retrieve top-5 relevant chunks from pgvector for the latest user message (filter by `universityId` + optional `subjectId`)
- Build prompt: system context (retrieved chunks) + conversation history
- Stream LLM response back
- Save conversation to `AIConversation` model (simple JSON log)

Student page: chat UI with message history. Subject selector to narrow the RAG scope.

---

### Quiz Generator (Django)

`POST /quiz/generate` вЂ” input: `{ subjectCode, topic, questionCount: 10 }`
- Prompt LLM with notes context + topic to generate MCQs in strict JSON format:
```json
{
  "questions": [
    { "text": "...", "options": [{"id":"A","text":"..."},...], "correctOption": "B", "explanation": "..." }
  ]
}
```
- Return to Express в†’ Faculty sees draft questions in editor
- Faculty edits/deletes questions в†’ saves to `Quiz` + `Question` tables with `isAiGenerated: true`

---

### PYQ Analysis (Django)

`POST /pyq/analyze` вЂ” input: `{ subjectId, fileUrls: [...] }` (HOD uploads PYQ PDFs)
- For each PDF: extract text в†’ extract question sentences в†’ cluster by topic using embeddings + k-means or LLM classification
- Build topic frequency map: `{ "Pipelining": 8, "Cache Memory": 5, ... }`
- POST back to Express: store in a `PYQAnalysis` table `{ subjectId, semesterId, topicFrequencies: JSON }`

Student sees: "High priority topics" card on study planner sorted by frequency. Study planner auto-suggests allocation: high frequency topic = more days.

- Calendar/week view (existing UI)

- Django generates a suggested weekly plan (`{ date, subject, topic, hours }`)
- Student can drag/drop to adjust, plan saved as JSON in student's profile

### Year Promotion Engine

This is the most complex feature. It solves the problem: Kavy is in `FY-3` (batch B3, sem 1) and needs to be moved to `SY-2` (batch C2, sem 3) in the new academic year.

**Rule: FY students в†’ SY. SY students в†’ TY. TY students в†’ FINAL. FINAL students в†’ alumni (inactive).**

**But the target batch is never automatic** вЂ” HOD decides which FY student goes to which SY batch (because branches reshuffle).

**Backend вЂ” Promotion API**

`GET /hod/promotion/preview?fromAcademicYearId=&toAcademicYearId=`
- Returns all currently active students grouped by their current batch + year level
- Returns available target batches in the destination academic year

`POST /hod/promotion/execute` вЂ” body:
```json
{
  "fromAcademicYearId": "uuid",
  "toAcademicYearId": "uuid",
  "mappings": [
    {
      "studentId": "uuid",
      "fromEnrollmentId": "uuid",
      "toSemesterId": "uuid",
      "toBatchId": "uuid",
      "toRollNo": "IT-25-042"
    }
  ]
}
```
Service logic (`services/promotion.service.ts`):
1. Validate: `toSemesterId` must belong to `toAcademicYearId`
2. For each mapping:
   - Set old `StudentEnrollment.isCurrent = false`
   - Create new `StudentEnrollment` with `{ studentId, semesterId: toSemesterId, batchId: toBatchId, rollNo: toRollNo, yearLevel: newYearLevel, promotedFromId: fromEnrollmentId, isCurrent: true }`
3. Return: `{ promoted: N, skipped: M }`

Students not in the mapping are NOT promoted (they may be detained вЂ” HOD can re-upload them later or handle separately).

**Bulk Roll Number Assignment**

HOD can upload a CSV for roll number assignment during promotion:
```
enrollment_no, new_roll_no, new_batch_code
LJ20IT001, IT-25-042, C2
```
System resolves `new_batch_code` to `batchId` in the target academic year.

**Frontend вЂ” Promotion Wizard (5 steps)**

Step 1: Select source year + target year (must be created first)
Step 2: View source students grouped by current batch
Step 3: Drag students to target batches (or upload CSV mapping)
Step 4: Assign roll numbers (manual or CSV)
Step 5: Preview + confirm в†’ call execute API

After promotion, the new `AcademicYear` can be set as active.

**Student History View**

`GET /student/history` (student-facing) вЂ” returns all enrollment rows ordered by semester number:
```json
[
  { "sem": 1, "batch": "B3", "yearLevel": "FY", "rollNo": "IT-23-042" },
  { "sem": 2, "batch": "B1", "yearLevel": "FY", "rollNo": "IT-23-019" },
  { "sem": 3, "batch": "C2", "yearLevel": "SY", "rollNo": "IT-24-057" }
]
```

`GET /hod/student/:enrollmentNo/history` вЂ” HOD view of any student's full journey.

## Cross-Cutting Concerns (Apply to All Phases)

### Error Handling
- Express: `errorHandler.ts` middleware catches all thrown errors
- Custom error classes: `ValidationError (400)`, `AuthError (401)`, `ForbiddenError (403)`, `NotFoundError (404)`
- All controllers `try/catch` в†’ `next(error)`
- Client: Axios interceptor catches errors, shows toast via `react-hot-toast`

### Pagination
- All list endpoints: `GET /resource?page=1&limit=20`
- Response shape: `{ data: [], total: N, page: 1, limit: 20, totalPages: N }`
- `utils/paginate.ts`: `prisma.model.findMany({ skip: (page-1)*limit, take: limit })`

### Soft Deletes
- `deletedAt DateTime?` on Student, Faculty, Note
- Prisma middleware to auto-filter `WHERE deletedAt IS NULL` on all queries
- Admin-only hard delete endpoint if needed

### Data Isolation
- Every Prisma query that touches university-scoped data includes `universityId: req.university.id`
- No cross-tenant data ever leaks вЂ” validated in tests

### Logging
- `pino` logger in Express (structured JSON logs)
- Log levels: `info` for requests, `warn` for auth failures, `error` for exceptions
- Production: logs shipped to CloudWatch

### Rate Limiting
- `express-rate-limit`: 100 requests/15min per IP on auth routes
- 1000 requests/15min per authenticated user on API routes

### Input Validation
- All request bodies validated with `zod` schemas in controllers
- `validateBody(schema)` middleware helper that returns 400 with field-level errors

## Key Design Decisions Summary

| Decision | Choice | Reason |
|---|---|---|
| Permanent student ID | `enrollment_no` (college-issued) | Never changes across 4 years. Roll no is semester-scoped. |
| Year promotion | Explicit HOD mapping (not auto) | Batches reshuffle вЂ” no algorithm can know which FY-3 student goes to which SY batch |
| HOD = Faculty | `isHod` flag on `Faculty` table | Simplifies auth (one table), HOD excluded from mentor assignments |
| Mentor chat history | Persisted in DB | Students and faculty need to reference old conversations |
| AI as separate service | Django on port 8001 | Python ecosystem (LangChain, PyMuPDF, Celery) doesn't mix with Node runtime |
| Multi-tenant isolation | `universityId` on every query | Row-level isolation, no schema-per-tenant complexity |
| Attendance cache | Redis with 5-min TTL | Computed % is expensive for 1000+ students, queried frequently |
| File storage | AWS S3 | Cheap, reliable, presigned URLs for direct client download |
| Soft deletes | `deletedAt` column | Academic records must be preserved for accreditation |
